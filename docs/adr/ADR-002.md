# ADR-002: Database Isolation

Status: Proposed for R0 Gate

Source: `docs/package/codex/01_Adopted_Decisions_ADR.md` ADR-002. Trace: DEC-02, C-7.

## Context

Cross-tenant leakage is a critical failure mode. Schema-per-tenant and database-per-tenant add too much operational complexity for the initial team and release path.

## Decision

Start with a shared PostgreSQL database. Every row-level tenant table must include `tenant_id NOT NULL` and RLS in the same migration. Exceptions such as `tenants` require an explicit migration comment. Tenant context must be centralized, and future schema-per-tenant expansion must not be blocked.

## Consequences

RLS provides a database-level guard, while application queries still must inject permission filters before execution. Every migration must prove tenant isolation through negative tests, and cross-tenant tests remain release-gate evidence.

## Review Triggers

Revisit for contractual schema/DB-per-tenant requirements, proven RLS performance failure, or R13 enterprise hardening.
