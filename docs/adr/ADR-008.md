# ADR-008: Queue and Worker Strategy

Status: Accepted

Source: `docs/package/codex/01_Adopted_Decisions_ADR.md` ADR-008. Trace: DEC-06, DEC-08.

## Context

Document ingestion needs asynchronous processing, but adding Kafka or Redis before core governance is stable increases operational and permission-audit complexity.

## Decision

Use pg-boss on PostgreSQL as the initial queue. Python ingestion workers process assigned jobs only and return parsed artifacts through approved NestJS-side services. Kafka, Redis, or other queue/cache systems are deferred until R4 or later reevaluation.

## Consequences

The queue shares backup and deployment boundaries with PostgreSQL. Throughput is sufficient for early releases, and job payloads must avoid sensitive body logging. Any worker output that changes persisted state must preserve audit-by-default.

## Review Triggers

Revisit if ingestion throughput exceeds pg-boss capacity, if queue reliability requires a separate broker, or during the R4 queue/cache reevaluation.
