# ADR-004: Permission Model

Status: Proposed for R0 Gate

Source: `docs/package/codex/01_Adopted_Decisions_ADR.md` ADR-004. Trace: DEC-09, DEC-15, C-1, C-2, C-7.

## Context

The source package defines broad permission layers, but a full ABAC launch would make evaluation opaque and hard to verify. Permission-filter omissions are critical risks.

## Decision

Use seven v1 roles with one `practice_group` attribute. Permission evaluation is default-deny, deny-overrides, requires `matter_members` for allow, and rejects unparseable `condition_json`. All authorization goes through PermissionService. External User remains inactive before R11. Permission Model Freeze is required after R1 before R2 or R3 work.

## Consequences

Document and search endpoints may not bypass PermissionService. Tests must include unauthorized negative cases. ABAC-like needs before R5 must be represented through explicit deny or walls, or escalated.

## Review Triggers

Revisit in R5 for ABAC generalization, in R11 for external role activation, and in R13 for SSO/SAML subject-model changes.
