# ADR-007: Storage and Encryption

Status: Accepted

Source: `docs/package/codex/01_Adopted_Decisions_ADR.md` ADR-007. Trace: DEC-07, DEC-13.

## Context

Original documents live in object storage, while structured metadata lives in PostgreSQL. Storage compromise and sensitive identifier exposure are high-impact risks.

## Decision

Use S3-compatible object storage with MinIO for development. Storage paths require a tenant prefix through a resolver, not hand-built strings. At-rest encryption is default. Column-level encryption is reserved for the five DLP identifier classes named in the package.

## Consequences

The system remains portable across S3-compatible targets. Original file objects are immutable and versioned. Encryption hooks can evolve toward KMS/BYOK without changing domain behavior.

## Review Triggers

Revisit during R2 object-storage implementation, when KMS/BYOK requirements are approved, or when DLP policy scope changes.
