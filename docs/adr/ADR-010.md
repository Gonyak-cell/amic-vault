# ADR-010: API Versioning

Status: Accepted

Source: `docs/package/codex/01_Adopted_Decisions_ADR.md` ADR-010. Trace: DEC-14.

## Context

The API will evolve across many releases and must avoid silent contract breaks for clients, tests, and future integrations.

## Decision

Use a URL prefix versioning scheme with `/v1` for initial APIs. New endpoints must live under the versioned prefix unless an approved ADR update defines another boundary.

## Consequences

Routing, documentation, and client contracts all have a stable version anchor. Breaking changes require a deliberate versioning decision rather than ad hoc endpoint edits.

## Review Triggers

Revisit when a breaking API change is unavoidable or when external integration requirements demand a different versioning strategy.
