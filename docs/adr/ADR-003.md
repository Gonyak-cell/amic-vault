# ADR-003: Runtime Split and Monorepo

Status: Proposed for R0 Gate

Source: `docs/package/codex/01_Adopted_Decisions_ADR.md` ADR-003. Trace: DEC-03, DEC-04, DEC-06.

## Context

Domain APIs need TypeScript type safety and shared frontend contracts, while OCR, HWPX, and document parsing need the Python ecosystem.

## Decision

Use a pnpm/turborepo monorepo. Implement domain backend APIs in NestJS, the frontend in Next.js/React, shared contracts in `packages/shared`, pure domain rules in `packages/domain`, and ingestion in a Python 3.12 FastAPI worker under `workers/ingestion`.

## Consequences

TypeScript packages are verified by `pnpm lint`, `pnpm typecheck`, `pnpm test`, and `pnpm build`. The Python worker is a separate runtime and must never become a permission or audit bypass; persisted effects flow through NestJS services.

## Review Triggers

Revisit if TypeScript parsing quality becomes equivalent to Python for required legal-document formats, or if worker throughput requires a different runtime topology.
