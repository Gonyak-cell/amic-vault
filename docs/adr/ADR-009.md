# ADR-009: HWP/HWPX Strategy

Status: Accepted

Source: `docs/package/codex/01_Adopted_Decisions_ADR.md` ADR-009. Trace: DEC-10.

## Context

Korean legal workflows require HWP-family support, but binary HWP 5.0 parsing is riskier than HWPX XML text extraction.

## Decision

Implement HWPX XML text extraction in R2. Treat binary HWP 5.0 as a later dedicated track for R4 through R6. Do not add unsupported binary parsing shortcuts in early releases.

## Consequences

Early ingestion can support the safer XML format while avoiding brittle binary parser commitments. Unsupported HWP 5.0 inputs must fail clearly and auditably until the approved track lands.

## Review Triggers

Revisit when R4-R6 binary HWP support is scheduled or when customer import requirements make binary HWP mandatory.
