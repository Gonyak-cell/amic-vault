# ADR-006: Search Engine Phasing

Status: Accepted

Source: `docs/package/codex/01_Adopted_Decisions_ADR.md` ADR-006. Trace: DEC-05, C-5.

## Context

Adding PostgreSQL FTS, OpenSearch, vector indexes, and graph storage too early would multiply permission-filter surfaces before the permission model is frozen.

## Decision

Use PostgreSQL 16 FTS for R3 keyword search. Decide OpenSearch only at the R3 Gate based on Korean-tokenization evidence. Vector search uses pgvector no earlier than R6. Neo4j and GraphSync are prohibited before R7. Search permissions must be injected at query time, not filtered after retrieval.

## Consequences

R3 infrastructure remains PostgreSQL-centered. Search quality tradeoffs are accepted until measured. Any later engine transition must re-prove permission filtering, tenant isolation, metadata leakage controls, and index synchronization.

## Review Triggers

Revisit at the R3 search evaluation gate, on measured PG FTS SLA failure, during R6 hybrid retrieval planning, and during R7 graph planning.
