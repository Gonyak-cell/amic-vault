# ADR-005: Audit Immutability

Status: Proposed for R0 Gate

Source: `docs/package/codex/01_Adopted_Decisions_ADR.md` ADR-005. Trace: DEC-12, Brief Constitution.

## Context

Audit logs are evidentiary infrastructure for legal systems. Mutable audit logs or sensitive body text in metadata would make the audit layer itself a risk.

## Decision

`audit_events` is append-only. Database permissions and triggers must block UPDATE and DELETE. Corrections are new events. Metadata is whitelist-only and must contain references, IDs, or hashes rather than confidential source text.

## Consequences

Actions that require audit events are incomplete if audit recording fails. Storage grows monotonically until approved records-management policy arrives, and schema changes must be additive.

## Review Triggers

Revisit during R12 records-management policy work, when storage costs require archive design, or when SIEM/WORM requirements appear in R13.
