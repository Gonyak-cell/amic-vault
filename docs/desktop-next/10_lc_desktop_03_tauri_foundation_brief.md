# 10. LC-DESKTOP-03 Tauri Foundation Brief

Date: 2026-06-21
Status: Implemented and verified after operator dependency approval.

## 1. Goal

Create the smallest `apps/desktop` Tauri v2 thin-shell scaffold for AMIC Vault while preserving server authority and avoiding local data storage.

This LC must not make AMIC Vault a desktop-hosted vault. It may only create a native access surface over an approved Vault web origin.

## 2. Dependency Gate

LC-DESKTOP-03 was blocked until an operator or reviewer explicitly approved adding Tauri/Rust desktop dependencies in this branch. The operator approved the gate in the active Codex thread:

```text
Approve LC-DESKTOP-03 dependency gate for a minimal Tauri v2 thin-shell scaffold: yes.
```

Allowed dependency family after approval:

- Tauri v2 CLI and app packages needed for a minimal desktop shell.
- Rust crates generated by the Tauri v2 scaffold only when they do not add database, storage, model, search, vector, OCR, file-sync, or local-vault runtime behavior.

Disallowed dependency family:

- Electron.
- Local database engines.
- Vector stores or embedding libraries.
- AI or LLM clients.
- Search index engines.
- Storage clients used directly by desktop.
- External sharing, mail, or M365/Outlook integration packages.

No package may be added solely because a scaffold template suggests it. Each dependency must be visible in the LC-DESKTOP-03 executor evidence.

- `packages/ai/**`
- `workers/ingestion/**`
- `db/migrations/**`
- server permission, audit, search, AI, records, storage, or auth behavior
- release production deploy gates

## 6. Required Scaffold Properties

The scaffold must:

- define package name `@amic-vault/desktop`;
- document that it is not a local Vault runtime;
- not hard-code staging or production private origins;
- keep updater disabled;
- keep native capabilities denied by default;
- avoid filesystem, clipboard, dialog, shell-open, share, mail, scanner, folder watch, OCR, or local AI features;
- avoid any local database or local document/search/AI/audit cache;
- be able to fail closed when approved-origin config is missing.

## 9. Manual QA Surface

If platform prerequisites are available:

- launch a debug shell;
- confirm missing approved-origin config blocks or renders only a non-sensitive safe state;
- confirm no local server, database, worker, storage, search, vector, or AI process starts.

If platform prerequisites are missing:

- record the missing prerequisite exactly;
- verify static scaffold files and capability config;
- do not claim native runtime QA as passed.

## 10. Stop Conditions

Stop immediately if:

- dependency approval is missing;
- Electron appears in the diff;
- server runtime, DB, storage, worker, search, vector, model gateway, or document database is bundled;
- private origin, token, cookie, AWS account ID, signing secret, or customer data appears in the repo;
- any broad native capability is opened by default;
- `docs/package/**` would need modification.
